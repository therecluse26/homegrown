## 1 · Critical (P0) — Legal / Security / Data Loss

### CRIT-1 · NoopThornAdapter — no CSAM detection or NCMEC reporting

`internal/safety/noop_adapters.go` — `NoopThornAdapter{}` remains:

- `ScanCsam` always returns `&CsamScanResult{IsCSAM: false}` — no content is ever flagged.
- `SubmitNcmecReport` returns `nil, nil` — no report is ever filed.
- `CheckHashUpdate` returns `false, nil` — hash database never refreshed.

Note: `safety.LoggingThornAdapter{}` is used in the worker registration
(`cmd/server/main.go:1287`), which logs warnings but still does not perform real scans.

**Impact:** Federal legal obligation under 18 U.S.C. § 2258A requires electronic service
providers to report CSAM to NCMEC. A noop adapter means the platform cannot comply.

**Carried forward from:** core-gaps.md CRIT-2 (unchanged).

---

### CRIT-2 · Lifecycle job workers not registered — export/deletion jobs silently dropped

`cmd/server/main.go` registers task handlers for `notify`, `recs`, `comply`, `learn`,
`media`, `social`, and `safety` — but **not lifecycle**. The lifecycle service enqueues two
job types:

- `lifecycle:data_export` (via `DataExportJob{}`)
- `lifecycle:process_deletion` (via `ProcessDeletionJob{}`)

These jobs enter the Asynq queue but no worker ever picks them up. They accumulate in the
queue silently until they hit the dead-letter threshold.

**Impact:** Data export requests and deletion requests appear to succeed (HTTP 202) but
never actually execute. GDPR Article 17 and COPPA §312.10 compliance is broken at runtime.

**New finding.** The lifecycle domain itself is now fully wired (handler, repository,
migration 25, AppState registration) — but the background worker gap makes it
non-functional.

### CRIT-3 · Lifecycle ExportHandler / DeletionHandler lists empty

`cmd/server/main.go:1127-1128`:
```go
[]lifecycle.ExportHandler{},   // domain export handlers — registered at startup in Phase 2
[]lifecycle.DeletionHandler{}, // domain deletion handlers — registered at startup in Phase 2
```

Even if CRIT-2 is fixed (workers registered), the export job calls

**Impact:** Export archives are empty. Deletions complete without deleting anything.
Combined with CRIT-2, the entire lifecycle domain is a no-op despite being fully wired.

**New finding.**

### CRIT-4 · COPPA micro-charge stub accepts any token

`internal/iam/service.go` — `SubmitCoppaConsent` validates that `method` and `token` are
non-empty but performs no actual Stripe micro-charge. A `TODO(billing::)` marks the

**Impact:** Credit card micro-charge is an FTC-recognized COPPA parental consent method.
The current stub accepts fabricated tokens as valid consent.

**Carried forward from:** core-gaps.md §2 01-iam (unchanged).

### P1-3 · Lifecycle deletion flow swallows session/subscription errors

`internal/lifecycle/service.go` `processSingleDeletion()` (lines ~348-391):

```go
if err := s.iamSvc.RevokeFamilySessions(ctx, req.FamilyID); err != nil {
    slog.Error("lifecycle: revoke family sessions", ...)
    // continues — error not tracked
}
if err := s.billingSvc.CancelFamilySubscriptions(ctx, req.FamilyID); err != nil {
    slog.Error("lifecycle: cancel subscriptions", ...)
    // continues — error not tracked
}
```

Session revocation and subscription cancellation failures are logged but not tracked in
`allSucceeded`. If both fail but all domain handlers succeed, the deletion is marked
"completed" — leaving active sessions and running subscriptions for a "deleted" family.

Note: Domain handler errors ARE properly tracked via `allSucceeded` and
`UpdateDomainStatus`. Only the pre-handler steps are affected.

**Impact:** Partially deleted families may retain active sessions and billing subscriptions.

**New finding (scope narrower than initially estimated).**

## 8 · Prioritized Remediation Roadmap

### P0 — Legal / Compliance (before any public beta)

   a logging adapter that queues manual review). 18 U.S.C. § 2258A compliance.
4. **Wire COPPA micro-charge** (CRIT-4) — implement the Stripe/Hyperswitch micro-charge in
   `SubmitCoppaConsent`.

   handler endpoints missing validation. Priority: comply (22), learn (25), admin (12).

   for payout aggregation and execution.
7. **Track lifecycle pre-handler errors** (P1-3) — propagate session revocation and
   subscription cancellation failures into the deletion status tracking.

   comply domain's cross-domain interfaces to unblock portfolio generation.

    media URL.
