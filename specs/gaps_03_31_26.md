## 1 · Critical (P0) — Legal / Security / Data Loss

### CRIT-1 · NoopThornAdapter — no CSAM detection or NCMEC reporting

`internal/safety/noop_adapters.go` — `NoopThornAdapter{}` remains:

- `ScanCsam` always returns `&CsamScanResult{IsCSAM: false}` — no content is ever flagged.
- `SubmitNcmecReport` returns `nil, nil` — no report is ever filed.
- `CheckHashUpdate` returns `false, nil` — hash database never refreshed.

Note: `safety.LoggingThornAdapter{}` is used in the worker registration
(`cmd/server/main.go:1287`), which logs warnings but still does not perform real scans.

**Impact:** Federal legal obligation under 18 U.S.C. § 2258A requires electronic service
providers to report CSAM to NCMEC. A noop adapter means the platform cannot comply.

**Carried forward from:** core-gaps.md CRIT-2 (unchanged).

### FE-6 · Hand-written API types in frontend hooks

Three hook files define hand-written TypeScript types instead of importing from
`@/api/generated/schema` (§3.1):

- `frontend/src/hooks/use-recommendations.ts` — `Recommendation`, `RecommendationPreferences`
- `frontend/src/hooks/use-search.ts` — 13 interfaces (search results, facets, params)
- `frontend/src/hooks/use-subscription.ts` — `Subscription`, `PaymentMethod`, `Transaction`, etc.

**Root cause:** The backend swag annotations for recs, search, and billing do not yet produce
response shapes that match what the frontend consumes. Running `make full-generate` does not
produce these types in `schema.ts`.

**Fix:** Add/correct swag annotations on recs, search, and billing handlers so that
`make full-generate` produces the matching types, then replace hand-written interfaces with
generated imports.

**Severity: P2** — functional but violates §3.1 type provenance rule.
